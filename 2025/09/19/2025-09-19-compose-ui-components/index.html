Jetpack Compose 实现字幕条、单词弹窗、句子面板、对话面板

Created2025-09-19|Updated2026-06-07|电影英语助手
|Post Views:

天气与目标

九月中旬，秋意渐浓。手机屏幕上方飘着一个悬浮窗，里面要能看书、能查词、能聊天——这不是一个 App，是四个面板拼起来的。

今天的目标：用 Jetpack Compose 实现电影英语助手的四个核心 UI 组件。分别是字幕条（SubtitleBar）、单词弹窗（WordPopup）、句子分析面板（SentencePanel）、llama 对话面板（LlamaChatPanel）。每个组件都独立可复用，组合在一起就是一个完整的”边看边学”界面。

（代码文件路径：project/app/src/main/java/com/movieenglish/assistant/floating/{SubtitleBarComposable.kt, WordPopupComposable.kt, SentencePanelComposable.kt, LlamaChatComposable.kt}）

原理

四个组件的关系

技术选型

四个组件全部使用 Jetpack Compose + Material3，不混用旧的 View 系统。原因：

  1. 声明式 UI 天然适合”数据驱动”的字幕更新（AnnotatedString 直接由数据状态决定颜色）
  2. LazyColumn 让聊天消息列表天然支持虚拟滚动
  3. Card + Surface 提供了一致的暗色主题容器
  4. Modifier 链式调用 让布局代码紧凑可读

实现步骤

第一步：SubtitleBar – 底部字幕条

这个组件是用户看得最多的。核心功能：在字幕文本中用 AnnotatedString 给不同类别的词染上不同颜色。

第二步：WordPopup – 单词详情弹窗

用户点击字幕中的黄色难词，弹出一个卡片，展示音标、含义、词缀拆解、同类词。

关键设计决策：MorphologyRow 不是硬编码的颜色，而是将词缀类型映射到不同颜色——用户一眼就能看出一个词的构成：”哦，原来 ‘incredible’ 是 in-（不）+ cred（相信）+ -ible（可…的）= 难以置信的”。这是长期积累的词汇学习方法论落地到 UI 上。

第三步：SentencePanel – 句子分析面板

用户点击红点后弹出的卡片，展示整句分析。这一屏的信息密度最高。

第四步：LlamaChatPanel – AI 对话面板

这是最复杂的组件——底部弹出的聊天界面，有消息列表、上下文展示、输入框。

可能遇到的问题

问题 1：AnnotatedString 对中文文本不起作用

buildHighlightedText 依赖空格分词，如果字幕里混了中文（如翻译字幕），split(Regex("(?<=\\s)|(?=\\s)")) 会把中文每个字都分开。

解决：在构造 AnnotatedString 之前判断字幕语言，中文文本跳过着色逻辑。

问题 2：LazyColumn 滚动位置不跟随新消息

LlamaChatPanel 中，当 llama 返回新消息时，LazyColumn 不会自动滚到底部。

解决：使用 rememberLazyListState() 并在消息列表变化时调用 listState.animateScrollToItem(messages.size - 1)。

问题 3：多个弹出面板的层次管理

WordPopup、SentencePanel、LlamaChatPanel 可能同时可见（虽然不推荐），需要做 z-order 管理。

解决：在 FloatingWindowService 中维护一个”当前激活面板”状态，同一时间只显示一个面板。点击红点关闭弹窗、点击弹窗外的区域也关闭。

小结

四个 Compose 组件构成了电影英语助手的完整 UI 层。核心思路是 AnnotatedString 负责着色，Card 负责容器，LazyColumn 负责列表。Compsoe 声明式的写法让”数据变了 UI 自动变”这个需求变得自然——字幕换一句，WordLevelModel 更新，列表重组，颜色就变了，不需要手动 invalidate。
