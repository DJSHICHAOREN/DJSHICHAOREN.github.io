为什么用 KSP 而不是 kapt？

KSP（Kotlin Symbol Processing）是 Google 专门为 Kotlin 设计的注解处理器。Room 的 @Entity、@Dao、@Database 注解在编译时需要生成实现类，KSP 比 kapt 快 2 倍，且不依赖 Java 注解处理框架。

常见坑：如果用 annotationProcessor 声明 Room compiler，Kotlin 文件中的 Room 注解不会被处理，编译能过但运行时会崩溃。Kotlin 项目必须用 kapt 或 KSP。

可能遇到的问题

问题 1：Unresolved reference: compose

原因：buildFeatures { compose = true } 没加。

解决：在 android {} 块中加上。

问题 2：Room 注解不生成代码，运行时 AbstractMethodError

原因：用了 annotationProcessor 而不是 ksp 或 kapt。

解决：改成 ksp("androidx.room:room-compiler:2.6.1")，并在根目录 build.gradle.kts 加上 KSP 插件声明。

问题 3：Theme.MovieEnglish not found

原因：Manifest 中引用的 @style/Theme.MovieEnglish 没有定义。

解决：创建 res/values/themes.xml 并定义同名主题。

问题 4：INSTALL_FAILED_NO_MATCHING_ABIS

原因：在 x86 模拟器上安装了只含 arm64-v8a 的 APK。

解决：用真机测试，或创建 x86 模拟器时加 abiFilters += "x86_64"。
