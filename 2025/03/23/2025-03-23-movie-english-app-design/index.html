天气与目标

今天窗外下着小雨，适合坐下来把想法捋清楚。

我决定做一个 Android App，它的功能很简单：你在手机上打开 Netflix 或本地播放器看电影，屏幕底部出现一条半透明字幕条，自动识别画面里的英文字幕，标出你不认识的高难度词汇；点击单词可以看解释和词根词缀拆解，点击句子开头的红点看到整句翻译和语法分析；遇到怎么都想不通的句子，上滑展开一个对话框，直接跟内置的 AI 语言模型对话请教。

今天的目标：把这个想法变成一份能落地的架构设计文档。

为什么要做这个 App？

很多人喜欢通过看电影学英语，但实际操作起来很痛苦：

  1. 字幕一闪而过，来不及查词
  2. 切到词典 App 查词，电影被打断
  3. 很多词是俚语、口语，常规词典查不到
  4. 知道单词意思但看不懂整句——语法结构不懂
  5. 不知道哪些词应该重点学——屏幕上词汇太多

市面上有一些 App 做字幕翻译，但它们都有一个致命问题：依赖网络。在地铁上、飞机上、国外旅行时，没网就废了。

所以我决定做一个完全离线的解决方案——OCR 模型本地识别字幕，AI 语言模型本地运行在手机上。

关键技术决策

决策 1：模型全内置 vs 按需下载

方案 优点 缺点
全内置 安装即用，无网络依赖 APK 体积 1.5-2GB
按需下载 安装包小 首次需联网，增加下载模块

选了全内置。 因为目标场景就是离线使用，一步到位。

决策 2：OCR 模型架构选 CRNN

为什么不用 Transformer 类模型（如 TrOCR）？因为 CRNN 足够小，量化后只有 3MB，手机推理 <50ms。TrOCR 动辄几百 MB，不适合嵌入。

决策 3：语言模型选 Qwen 2.5 1.5B

候选 中文能力 体积 速度
Llama 3.2 1B 弱 小 快
Gemma 2 2B 一般 中 中
Phi-3.5 mini 3.8B 弱 大 慢
Qwen 2.5 1.5B 强 1.2GB(Q4) 10-20 tok/s

解释单词时需要用中文表达，Qwen 是唯一中英双母语的选项。

决策 4：屏幕捕获用 MediaProjection API

Android 获取其他 App 画面只有两条路：

  • MediaProjection：系统弹窗授权，合法稳定
  • AccessibilityService：无感获取，但 Google Play 政策收紧

选了 MediaProjection 作为默认方案，AccessibilityService 作为可选增强。

悬浮窗 UI 设计

交互规则：

  • 点击黄色单词 → 弹出解释气泡（音标 + 释义 + 词根词缀）
  • 点击 🔴 红点 → 弹出整句面板（翻译 + 逐词 + 语法）
  • 上滑字幕条 → 展开 llama 对话面板
  • 右滑 → 临时隐藏

用户词库自适应算法

核心思想：用户的行为是最真实的反馈。

小结

这篇文章把想法变成了架构图、数据流图、决策表。

这个项目的代码量不小——4 个子系统，30 多个源文件。接下来我会一篇一篇记录每个模块是怎么做出来的，每一篇都包含原理 + 可执行的步骤 + 代码详解 + 踩过的坑。

如果你也在做一个类似的 App，或者想在 Android 上跑深度学习模型，这个系列也许能帮你少走一些弯路。
